# Общие правила мониторинга

Python фиксирует факты и пороговые отклонения. Причина отклонения не считается
подтвержденной без диагностических проверок.

Ожидаемое событие можно классифицировать как `expected_event` или
`expected_process_feature` только при подтверждении документацией или календарем.
Если подтверждения недостаточно, требуется `needs_manual_review`.

Для потенциальных инцидентов нужно отделять:

- факты из alert group;
- гипотезы возможных причин;
- конкретные рекомендуемые проверки.
