# Collection Monitoring Agent

## Что это

Демонстрационный AI-агент для мониторинга процессов Collection. MVP на
синтетических агрегированных данных находит отклонения, отличает ожидаемые
события от потенциальных инцидентов, проводит управляемую диагностику и
формирует готовый HTML-отчет.

## Бизнес-проблема

Регулярный мониторинг Collection содержит множество метрик по процессам,
моделям, качеству данных и бизнес-оптимизаторам. Аналитику приходится вручную
сопоставлять алерты, документацию и изменения показателей, чтобы понять:

- является ли отклонение ожидаемой особенностью процесса;
- где действительно возник потенциальный инцидент;
- какие проверки нужно выполнить в первую очередь.

## Идея решения

**Python считает факты. LLM объясняет их бизнесовым языком.**

Python детерминированно рассчитывает метрики, алерты, группы, приоритеты и
результаты диагностических проверок. GigaChat использует найденный
RAG-контекст, выбирает разрешенные инструменты расследования и интерпретирует
полученные evidence.

## Архитектура

```text
Synthetic data
      ↓
Alert objects
      ↓
Alert groups
      ↓
Mini-RAG over Markdown docs
      ↓
Alert Commentator
      ↓
Investigation Agent
      ↓
Final Summarizer
      ↓
Human Report
```

## Что агентного в решении

- GigaChat Planner выбирает диагностические tools для каждой alert group.
- Python проверяет выбранные инструменты по whitelist и ограничивает их число.
- Tools выполняются детерминированно над агрегированными данными.
- GigaChat Summarizer интерпретирует только фактические tool results.
- LLM не исполняет произвольный код и не изменяет расчетную бизнес-логику.

## Демо-сценарии

1. `bank_unavailable_day` — подтвержденный ЕДН, ожидаемое событие.
2. `credit_card_batch_inflow` — ожидаемое пакетное поступление `credit_card`.
3. `model_gini_drop` — значимое падение Gini модели.
4. `empty_feature_vector` — рост доли пустых векторов признаков.
5. `new_high_risk_segment` — появление нового рискованного сегмента.
6. `optimizer_mass_reject` — массовые отказы бизнес-оптимизатора.

## Как запустить

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
.venv/bin/python run_full_demo.py
```

Перед запуском укажите учетные данные GigaChat в `.env`.

## Главный результат

```text
outputs/runs/demo_run/human_report.html
```

Отчет содержит итоговое резюме, статистику, шесть alert groups, графики,
LLM-комментарии, RAG-источники и результаты Investigation Agent.

## Ограничения MVP

- используются только синтетические агрегированные данные;
- retrieval реализован как TF-IDF mini-RAG;
- LLM-слой работает через GigaChat API;
- реальные клиентские данные не используются;
- production-интеграции и промышленный scheduler не реализованы.

## Возможное развитие

- подключение документации из Confluence;
- embedding или hybrid RAG;
- регулярный запуск по расписанию;
- интеграция с промышленным мониторингом;
- human-in-the-loop подтверждение инцидентов.
